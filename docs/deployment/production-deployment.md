# 生产部署建议

## 部署原则

生产环境应关闭默认演示密码、替换 JWT 密钥、使用独立数据库账号、启用 HTTPS，并对真实模型 API Key 做后端加密存储。

## 推荐架构

- 前端静态资源部署到 Nginx 或对象存储/CDN。
- Java 后端部署为独立容器或云服务器服务。
- Python AI 服务独立扩缩容。
- MySQL 使用云数据库或主从备份。
- Redis 用于缓存、限流、任务状态或会话扩展。

## 安全配置

- 修改 `.env` 中的 `JWT_SECRET`。
- 禁用或删除演示账号。
- 真实 API Key 不写入 Git，不写入前端，不在日志中输出。
- 对 `/api/**` 保持 JWT 鉴权。
- 限制数据库端口对公网暴露。

## 运维建议

- 后端和 AI 服务增加日志采集。
- 为 MySQL 做每日备份。
- 使用健康检查配合容器重启策略。
- 对大模型调用增加超时、重试、限流和费用监控。
