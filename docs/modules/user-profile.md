# 用户画像与学习偏好模块说明

## 1. 模块定位

用户画像和学习偏好共同描述学习者的长期特征、阶段性目标和内容生成偏好。当前阶段只做手动维护和查询，不调用 Python AI 服务，也不自动推断画像。

## 2. 用户画像

用户画像表为 `user_profile`，支持两类画像：

| 类型 | 存储方式 | 说明 |
|---|---|---|
| 全局画像 | `space_id = 0` | 描述用户整体学习背景和长期偏好 |
| 空间画像 | `space_id = learning_space.id` | 描述某个学习空间下的阶段目标和薄弱点 |

画像字段包括真实姓名、学校、专业、年级、学习目标、学科方向、基础水平、兴趣标签、薄弱点、目标考试、每周可用学习时间、可用时间段、输出风格和画像来源。

画像不存在时，查询接口返回空画像结构，`status` 为 `incomplete`，不会自动写入数据库。用户提交更新时执行 upsert。

## 3. 学习偏好

学习偏好表为 `learning_preference`，属于用户级配置。字段包括资源类型偏好、输出风格、内容长度、难度、语言、学习时间段、通知开关、知识图谱开关、测验开关和复习计划开关。

偏好不存在时返回默认值：

| 字段 | 默认值 |
|---|---|
| `outputStyle` | `markdown` |
| `contentLengthPreference` | `medium` |
| `difficultyPreference` | `medium` |
| `languagePreference` | `zh-CN` |
| `knowledgeGraphEnabled` | `true` |
| `quizEnabled` | `true` |
| `reviewPlanEnabled` | `true` |

默认偏好用于前端展示和后续 AI 提示词组装，只有用户主动更新时才写入数据库。

## 4. 数据隔离策略

画像和偏好接口均要求登录。全局画像和学习偏好按当前用户 `user_id` 查询；空间画像在读写前必须先校验学习空间归属于当前用户。

## 5. 后续衔接

后续 AI 对话、资源生成、多智能体任务和学习报告会读取用户画像和学习偏好，用于生成个性化提示词、选择内容长度和难度、决定是否生成知识图谱或测验。
