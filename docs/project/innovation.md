# 项目创新点

## 1. Java 主体与 AI 微服务分工清晰

项目不是简单调用大模型的聊天应用，而是将 Java 主后端作为业务中枢，Python AI 微服务作为能力插件。Java 负责用户、权限、数据库、任务、资源和日志，Python 负责模型调用、多智能体编排和 RAG。该分工体现完整软件系统设计能力。

## 2. 用户自定义模型 API

系统支持用户配置自己的模型服务，包括 Mock、OpenAI-compatible、DeepSeek-compatible 和 Ollama 等形式。API Key 加密存储，前端只展示脱敏信息，既满足可扩展性，也满足安全展示要求。

## 3. 多智能体协同生成学习资源

系统将学习资源生成拆分为规划、知识结构、内容生成、习题生成、案例生成、质量审查、辅导解释和评估反馈等多个智能体职责。前端可以展示每个智能体的执行步骤，使 AI 过程可解释、可演示。

## 4. 学习闭环数据沉淀

系统不仅生成资源，还保存学习空间、用户画像、对话、知识库、资源、路径、测验、掌握度和学习报告。数据库能够支撑从需求输入到学习反馈的完整闭环。

## 5. Mock 模式保障演示稳定性

比赛和课程验收现场可能无法使用真实 API Key。本项目从数据库、后端和 AI 服务层面预留 Mock 模型配置和 Mock 生成结果，保证核心流程在无外网、无真实密钥情况下仍可演示。

## 6. 数据库设计可答辩

项目数据库设计包含 22 张以上核心表，并提供 ER 图、索引设计、范式分析、典型 SQL 和逻辑外键说明。该设计可以直接支撑数据库课程实习答辩和软件杯项目说明。
