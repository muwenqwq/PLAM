# 项目需求说明

## 1. 项目背景

“基于大模型的个性化资源生成与学习多智能体系统开发”要求参赛团队建设一个能够结合用户学习目标、知识基础和学习表现，自动生成个性化学习资源的软件系统。本项目将其落地为面向高校课程学习、教师辅助教学和自学复习的智能学习平台。

项目本质是 Java 软件开发与数据库设计综合实践。AI 能力作为系统亮点，但系统主体必须体现完整的软件工程能力：清晰的业务建模、规范的数据库设计、稳定的后端接口、可用的前端交互、可演示的 Mock 模式和完整的项目文档。

## 2. 用户角色

1. 学生用户：创建学习空间，配置学习画像，上传学习资料，发起对话和资源生成任务，完成测验并查看报告。
2. 教师用户：创建课程场景，生成讲义、习题和案例任务，查看学习数据和资源质量。
3. 管理员用户：管理系统用户、角色、操作日志和演示数据。

## 3. 功能需求

### 3.1 用户认证与权限

- 支持用户注册、登录、退出和当前用户信息查询。
- 使用 BCrypt 存储密码，使用 JWT 完成接口鉴权。
- 支持角色预留，至少包括管理员、教师和学生。
- 重要接口必须进行用户数据隔离。

### 3.2 学习空间与画像

- 用户可以创建多个学习空间，每个空间对应一个课程、考试或学习目标。
- 用户可以维护学习目标、基础水平、兴趣方向、学习时间和资源偏好。
- 系统通过画像和偏好影响 AI 资源生成结果。

### 3.3 AI 模型配置

- 用户可以配置 Mock、OpenAI-compatible、DeepSeek-compatible、Ollama 等模型来源。
- 前端不得获取明文 API Key。
- 后端保存加密后的 `api_key_encrypted` 和脱敏展示字段 `api_key_masked`。
- 没有真实 API Key 时系统必须自动支持 Mock 演示。

### 3.4 智能对话与意图识别

- 用户可以通过自然语言描述学习需求。
- Java 后端保存对话与消息，并调用 Python AI 微服务获得回复。
- AI 微服务返回结构化意图、建议资源类型和后续任务参数。

### 3.5 多智能体资源生成

- 系统至少预留 PlannerAgent、KnowledgeAgent、ContentAgent、ExerciseAgent、CaseAgent、ReviewAgent、TutorAgent、AssessmentAgent。
- Java 后端创建任务、保存步骤、保存最终资源。
- Python AI 微服务执行 Mock 或真实模型编排，返回每个智能体的步骤摘要和结构化结果。

### 3.6 知识库与 RAG

- 支持用户上传 PDF、DOCX、PPTX、TXT、Markdown 等学习资料。
- Java 后端保存文件元数据和权限信息。
- Python AI 微服务负责解析、切分、检索和生成引用来源。

### 3.7 学习资源、测验和报告

- 支持生成学习计划、知识图谱、讲义、复习提纲、习题集、案例任务和 Markdown 学习手册。
- 支持在线测验、自动评分、错因分析和薄弱知识点识别。
- 支持学习报告、可视化统计和导出。

## 4. 非功能需求

- 可运行：本地环境至少支持 Java 后端、MySQL、前端和 Python AI 微服务分开启动。
- 可演示：Mock 模式下不依赖真实 API Key。
- 可部署：后续提供 Docker Compose 一键启动。
- 可维护：表名、字段名、接口名和代码结构保持规范稳定。
- 可扩展：AI 服务与主业务后端边界清晰，便于后续接入真实大模型。
- 可答辩：数据库设计、系统架构、创新点和演示流程有完整中文文档。

## 5. 当前阶段范围

第 0-1 阶段只完成项目结构、项目文档和数据库设计。不在本阶段实现 Spring Boot 业务代码、Vue 页面代码或复杂 Python AI 逻辑。第 2 阶段开始初始化 Java 后端基础工程。
